Disruptions

Different types of disruptions from devices

https://services.elemez.com/disruptions?from={{timestamp}}&to={{timestamp}}

This API from Elemez will return the total number of disruptions that have happened across all of your devices over the specificed time period.

You will be provided with the total number of disruptions per day over the time range, the mean, the standard deviation, and a measure of how much the latest value deviates from the mean.

Besides the trend values, the disruptions API also provides the following values:

mean
Arithmetic mean of the daily trend values (excluding the last day of the period)
stddev
Sample standard deviation of the daily trend values (excluding the last day of the period)
total
Number of disruptions on the last day of the time range (usually current day)
sigma
Signed integer indicating how many standard deviations the latest value deviates from the mean of the previous trend (rounded towards zero)

How other values can be used

Sigma

The sigma tells you how many standard deviations the latest value is from the mean. If the sigma value is high then the difference from the mean is also high. This value is supplied as a whole number that is rounded towards zero.

Mean

The mean value is a measure of the number of disruptions you see on an average day for the range selected. The mean is calculated for the selected range minus the last day of that range, which allows you to compare if the value for the latest day is different to the mean of the days before.

Standard deviation

The standard deviation represents the degree of variability present within a range of data when compared to the mean. If the standard deviation is low, this indicates that the data is close to the mean and that the variation is low. If the standard deviation is high, then the data has a wide range of values and is further away from the mean value.

Last Value

The Last Value is the number of disruptions on the last day of the selected range returned from the API. If you have chosen a date range which includes today, this will return the number of disruptions that have occurred today at the time of the request. As this is real time data, this figure is subject to change if the request is repeated or resent.

Integration

Our API is easy to use, below is an example of code that we used to get data from our servers: 

                
  function getDisruptions(callback) {
    var apigClient = apigClientFactory.newClient({apiKey: "elemez key value");
    var params = {
      'x-api-key': "elemez key value",
      token: "elemez token value",
      from: moment.utc().subtract(15, 'days').valueOf(),
      to: moment.utc().valueOf()
    };
    var body = {};
    var additionalParams = {};

    apigClient.disruptionsGet(params, body, additionalParams)
      .then(result => callback(result)).catch(result => {
        console.log("ERROR: ", result);
      });
  }
